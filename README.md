## What is COMPASS?

COMPASS is a comprehensive operating manual and protocol system designed specifically for AI-assisted software development. It establishes professional, security-first standards that ensure AI coding agents (Claude, Copilot, Cursor, ChatGPT, etc.) generate secure, well-tested, maintainable code that follows industry best practices.

Think of COMPASS as the **professional operating system for AI code generation** â€” transforming AI from a helpful assistant into a reliable engineering partner.

## Why COMPASS?

AI coding agents are powerful but inconsistent. Without clear standards, they may:

- Generate code with security vulnerabilities
- Skip tests or create inadequate test coverage
- Hallucinate dependencies that don't exist
- Over-engineer simple solutions
- Modify data destructively without backups
- Ignore existing codebase patterns

**COMPASS solves these problems** by providing:

âœ… **Security-first principles** â€” OWASP Top 10 compliance, input validation, secrets management
âœ… **Test-driven workflows** â€” Mandatory TDD, 70%+ coverage, systematic debugging
âœ… **Quality standards** â€” SOLID principles, clean architecture, maintainable code

âœ… **Professional workflows** â€” Git protocols, code review checklists, task management

## The 24 Core Principles

COMPASS is built on **24 operating principles** organized into 6 categories:

### 1. Security & Safety (P0-P3)
- **P0:** Security is mandatory and non-negotiable
- **P1:** Validate all inputs (allowlist patterns, parameterized queries)
- **P2:** Authentication and authorization for all sensitive operations
- **P3:** Never commit secrets or credentials

### 2. Code Quality & Testing (P4-P9)
- **P4:** Test-Driven Development (write tests first, 70%+ coverage)
- **P5:** Follow SOLID principles and clean architecture
- **P6:** Use descriptive names from the domain, not implementation details
- **P7:** Keep components focused (single responsibility, max 300 lines)
- **P8:** Modify code minimally (smallest change to achieve goal)
- **P9:** Write clear documentation (file headers, complex logic comments)

### 3. Development Workflow (P10-P13)
- **P10:** Use Git professionally (conventional commits, atomic changes)
- **P11:** Use TodoWrite for task tracking and progress visibility
- **P12:** Debug systematically (4-phase root cause analysis)
- **P13:** Verify before claiming completion (run commands, check output)

### 4. Data & State Management (P14-P16)
- **P14:** Back up data before destructive operations
- **P15:** Use local-first storage (IndexedDB, localStorage for offline capability)
- **P16:** Design for idempotency and safe retries

### 5. AI-Specific Behaviors (P17-P21)
- **P17:** Never hallucinate dependencies (search before suggesting)
- **P18:** Avoid over-engineering (YAGNI â€” You Aren't Gonna Need It)
- **P19:** Search codebase before creating new patterns
- **P20:** Match existing conventions and patterns
- **P21:** Ask user when uncertain rather than guessing

### 6. Operational Excellence (P22-P24)
- **P22:** Handle errors gracefully with proper recovery
- **P23:** Consider performance early (profile, then optimize)
- **P24:** Follow accessibility standards (WCAG 2.2)

## Example Workflows

### Example 1: Adding a New Feature with TDD

```markdown
1. Load COMPASS-TESTING.md
2. Write failing test first
3. Run test and confirm it fails
4. Implement minimal code to pass
5. Run test again and confirm it passes
6. Refactor for quality
7. Verify 70%+ coverage
8. Commit with conventional commit message
```

### Example 2: Debugging a Production Issue

```markdown
1. Load COMPASS-TESTING.md (systematic debugging section)
2. Phase 1: Root Cause Investigation
   - Gather error messages and logs
   - Identify the failure point
   - Trace backwards through call stack
3. Phase 2: Pattern Analysis
   - Check recent commits for similar issues
   - Review related test failures
4. Phase 3: Hypothesis Testing
   - Form specific hypothesis
   - Add instrumentation/logging
   - Test hypothesis with minimal changes
5. Phase 4: Implementation & Verification
   - Implement fix with test
   - Verify fix resolves issue
   - Add regression test
```
